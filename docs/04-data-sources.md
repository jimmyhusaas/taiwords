# 詞庫資料來源盤點

> 目標：建立一份**可長期維護、授權清楚、可被引用**的詞庫。
> 原則：寧可少而精準，不要多而錯亂。所有引用必須留下出處。

---

## 1. 收錄原則

1. **授權優先**：只收錄 CC、Apache、MIT、g0v 公共授權的資料；不爬商業辭典。
2. **可追溯**：每個詞條至少一個 `source_id`，不接受「不知道哪來的」。
3. **可比對**：兩岸對應詞要可被兩個獨立來源交叉驗證的，confidence 才能拉高。
4. **可審核**：政治／敏感詞必須過 `pending_review` 才能上線。

## 2. 主要候選來源

### 2.1 g0v moedict-data-csld（兩岸三地生活差異詞語彙編）
- **URL**：<https://github.com/g0v/moedict-data-csld>
- **授權**：沿用萌典開放精神（檔案內未明標，視為公共授權）
- **規模**：約 3,000 條對照詞
- **品質**：g0v 社群整理，可信度高
- **使用方式**：直接 fork、轉成本專案 schema、保留 `source_id=src-moedict-csld`
- **注意**：原始資料以「兩岸三地」為主，需重新標 confidence

### 2.2 OpenCC TWPhrases
- **URL**：<https://github.com/BYVoid/OpenCC>
- **授權**：Apache-2.0
- **規模**：數萬條繁簡 + 台灣慣用詞詞庫
- **品質**：工程界廣泛使用
- **使用方式**：抽取 `TWPhrases.txt` 作為「替換建議」基底，不直接判定支語
- **注意**：OpenCC 是「轉換」用途，未必每一條都是支語問題

### 2.3 大陸臺灣計算機術語對照表（維基教科書）
- **URL**：<https://zh.wikibooks.org/zh-tw/大陸臺灣計算機術語對照表>
- **授權**：CC-BY-SA-3.0
- **規模**：數百條 IT 術語
- **品質**：維基審核，但偏舊
- **使用方式**：補強 `it` 分類，並在 notes 中標註出處與條目連結
- **注意**：需依授權做出 CC-BY-SA-3.0 相容釋出

### 2.4 SeanPeng / HackMD 計算機術語對照
- **URL**：<https://hackmd.io/@SeanPeng/B1psY1s6K>
- **授權**：未明示
- **規模**：中型
- **使用方式**：**僅作為人工校對參考**，不直接匯入；要匯入須先聯絡作者
- **注意**：法律風險，預設不收錄

### 2.5 教育部重編國語辭典修訂本
- **URL**：<https://dict.revised.moe.edu.tw/>
- **授權**：教育部授權使用，需註明出處
- **使用方式**：作為「台灣慣用詞」的權威來源，引用釋義時連結回原網站
- **注意**：不批次爬，僅在編輯詞條時手動引用

### 2.6 PTT / Plurk / 噗浪 自建語料庫
- **URL**：N/A（自爬）
- **授權**：使用者公開貼文，學術／非商業使用一般可接受，但須匿名化
- **使用方式**：建立**測試集**（人工標註 ground truth），用來評估偵測器準確度
- **注意**：
  - 只保留**詞**與**上下文片段**，不保留使用者帳號
  - 學術 / 個人專案 fair use，**避免商用變現**
  - 若公開測試集，須移除可識別資訊

### 2.7 大陸網路用語整理（補強用）
- 「百度百科 / 知乎」**不爬**，避免授權與政治風險
- 改以**新聞報導**作為 secondary source（如「央視說某詞」），在 notes 中註明
- 中國時報、聯合報、自由時報的相關專題文章也可作為間接證據

## 3. 資料整併流程（pipeline）

```
[raw 來源檔]
   ↓ 1. 取得授權清單與原始檔
[normalized YAML]
   ↓ 2. 統一轉成本專案 schema（手寫 / 腳本）
[diff vs current DB]
   ↓ 3. 人工 review（policy：政治分類必須過審）
[Flyway migration]
   ↓ 4. 跑 CI 測試（schema check + 重複偵測 + confidence sanity）
[Production DB]
```

### 3.1 自動化 vs 人工
- 自動化能做的：去重、字串正規化、欄位完整性檢查、跨來源 cross-check
- **必須人工**的：confidence 評分、同名異實判定、政治分類審核

## 5. 授權合規檢查表（每次匯入前必填）

- [ ] 來源的授權是什麼？（明確的字串，例：`CC-BY-SA-4.0`）
- [ ] 是否需要在最終產品中標註出處？
- [ ] 是否允許**修改**？是否允許**商用**？
- [ ] 是否需要以**相容授權**釋出（copyleft）？
- [ ] 已在 `sources` table 留下 `id / url / license / type / description`
- [ ] 已在 `term_sources` 把每個詞條與來源連起來
